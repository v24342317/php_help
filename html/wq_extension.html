扩展相关函数

扩展相关函数全部位于 framework/model/extension.mod.php 文件内。

注意：该文件内所有函数使用前必须加载文件: load()→model('extension');

ext_module_checkupdate

boolean ext_module_checkupdate(string $modulename);

模块更新检测

参数
$modulename string 模块名称
返回值

如果模块需要更新，返回 true，否则返回 false。

示例
load()->model('extension');
$moduleInfo = ext_module_checkupdate('we7_car');
print_r($moduleInfo);    // true 需要更新

ext_module_bindings

array ext_module_bindings();

获取模块入口类型

参数

void

返回值

模块入口类型列表。

示例
 
load()->model('extension');
$moduleInfo = ext_module_bindings();
print_r($moduleInfo);
 
Array
(
    [cover] => Array
        (
            [name] => cover
            [title] => 功能封面
            [desc] => 功能封面是定义微站里一个独立功能的入口(手机端操作), 将呈现为一个图文消息, 点击后进入微站系统中对应的功能.
        )
 
    [rule] => Array
        (
            [name] => rule
            [title] => 规则列表
            [desc] => 规则列表是定义可重复使用或者可创建多次的活动的功能入口(管理后台Web操作), 每个活动对应一条规则. 一般呈现为图文消息, 点击后进入定义好的某次活动中.
        )

ext_module_clean

void ext_module_clean(string $modulename, boolean $isCleanRule);

模块信息删除

参数
$modulename string 模块名称
$isCleanRule boolean 是否删除相关规则
返回值

void

示例
 
load()->model('extension');
ext_module_clean('we7_car', true);    // 该函数执行后，将会删除数据库中有关 We7_car 模块信息。
